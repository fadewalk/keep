Keep 支持多种部署和配置策略，以适应不同的环境和使用场景，从开发环境到生产部署。

### 配置选项

Keep 提供三种主要的配置选项：

1. [**提供商配置**](/deployment/provision/provider) - 为 Keep 设置和管理数据提供商及其去重规则。
2. [**工作流配置**](/deployment/provision/workflow) - 在 Keep 中配置和管理工作流。
3. [**仪表板配置**](/deployment/provision/dashboard) - 在 Keep 中配置和管理仪表板。


选择合适的配置策略取决于您的具体使用场景、部署环境和可扩展性需求。您可以在各自的部分阅读更多关于每个配置选项的信息。

### 如何配置配置

<Tip>
  某些配置选项需要额外的环境变量。这些将在具体的配置页面详细说明。
</Tip>

Keep 中的配置通过环境变量和配置文件控制。主要的配置环境变量如下：

| 配置类型 | 环境变量 | 用途 |
| ---------------------- | ------------------------------ | ------------------------------------------------------------------------- |
| **提供商** | `KEEP_PROVIDERS` | 包含带去重规则的提供商配置的 JSON 字符串 |
| **工作流** | `KEEP_WORKFLOW` | 直接从环境变量配置的一个工作流 |
| **工作流** | `KEEP_WORKFLOWS_DIRECTORY` | 包含工作流配置文件的目录路径 |
| **仪表板** | `KEEP_DASHBOARDS` | 包含仪表板配置的 JSON 字符串 |

提示：使用以下脚本从工作流文件获取单行格式用于 KEEP_WORKFLOW：
```
使用 `cat workflow_file.yaml | awk '{printf "%s\\n", $0}' | tr -d '\n'; echo` 将工作流转换为字符串格式。
```

有关每种配置策略的更多详细信息，包括设置说明和影响，请参阅相应部分。
