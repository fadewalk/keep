## 传统告警与统一告警

Keep 支持 Grafana 的传统告警系统和较新的统一告警系统。以下是主要区别：

### 传统告警
- 使用通知渠道传递告警。
- 在仪表板级别配置。
- 使用不同的 API 端点（`/api/alerts` 和 `/api/alert-notifications`）。
- 设置更简单但功能较少。
- 告警与仪表板面板紧密耦合。

### 统一告警（Grafana 9.0 起默认）
- 使用告警规则和联系点。
- 在告警部分集中配置。
- 使用较新的 `/api/v1/alerts` 端点。
- 更强大的功能，包括基于标签的路由。
- 支持在单个告警规则中使用多个数据源。

<Note>
如果您使用的是 Grafana 8.x 或更早版本，或者在新版本中明确启用了传统告警，请确保使用传统告警配置相应地配置 Keep。
</Note>

## 连接 Provider

要连接到 Grafana，您需要创建一个 API 令牌：

1. 登录您的 Grafana 账户。
2. 进入 **Service Accounts** 页面（cmd+k -> service）。
3. 点击 **Add service account** 按钮并为您的服务账户提供一个名称。
4. 授予 "alerting" 权限：

**替代验证方法（当 Keep 无法从外部访问时）：**

如果 Keep 无法从外部访问且无法创建 webhook，您可以使用以下方法手动验证 Grafana provider 设置：

1. **在 Grafana 中手动测试告警：**
   - 在 Grafana 中创建一个手动测试告警。
   - 在 Grafana 中设置一个通常会向 Keep 发送告警的联系点。
   - 触发告警并检查 Grafana 的日志以获取错误或确认告警已发送。

2. **检查 Grafana 中的日志：**
   - 访问 Grafana 的日志文件或使用 **Explore** 功能查询与告警机制相关的日志。
   - 确保没有与 webhook 集成相关的错误，并且告警被正确处理。

3. **验证集成状态：**
   - 导航到 Grafana 中的 **Alerting** 部分。
   - 确认集成状态显示为活跃或正常运行。
   - 监控任何出站 HTTP 请求，以验证 Grafana 正在尝试与 Keep 通信。

4. **网络和连接检查：**
   - 使用网络监控工具确保 Grafana 可以到达 Keep 或为告警配置的任何替代端点。

## Webhook 集成修改

Webhook 集成将 Keep 添加为 Grafana 实例中的联系点。此集成可以在 "Contact Points" 部分找到。Keep 还可以访问以下范围：
- `alert.provisioning:read`
- `alert.provisioning:write`
