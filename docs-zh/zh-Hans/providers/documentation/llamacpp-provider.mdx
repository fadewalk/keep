<Tip>
  The Llama.cpp Provider supports querying local Llama.cpp models for prompt-based
  interactions. Make sure you have Llama.cpp server running locally with your desired model.
</Tip>

### **Cloud Limitation**
This provider is disabled for cloud environments and can only be used in local or self-hosted environments.

## 连接 Provider

要使用 Llama.cpp Provider：

1. 在您的系统上安装 Llama.cpp
2. 下载或转换您的模型为 GGUF 格式
3. 使用 HTTP 接口启动 Llama.cpp 服务器：
   ```bash
   ./server --model /path/to/your/model.gguf --host 0.0.0.0 --port 8080
   ```
4. 在 Keep 配置中配置主机 URL 和模型路径

## 前置条件

- 必须安装 Llama.cpp 并编译服务器支持
- 必须在您的系统上提供 GGUF 格式的模型文件
- Llama.cpp 服务器必须正在运行并可访问
- 服务器必须有足够的资源来加载和运行您的模型

## 模型兼容性

该 provider 可与任何与 Llama.cpp 兼容的 GGUF 格式模型配合使用，包括：
- LLaMA 和 LLaMA-2 模型
- Mistral 模型
- OpenLLaMA 模型
- Vicuna 模型
- 以及其他兼容的模型架构

在使用 provider 之前，请确保您的模型采用 GGUF 格式。
