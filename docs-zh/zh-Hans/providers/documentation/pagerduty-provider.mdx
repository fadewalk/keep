## 描述

Pagerduty Provider 实现了与 PagerDuty 的集成，以在 Keep 内创建、管理和同步事件和告警。它同时支持直接 API 密钥认证和 OAuth2，为安全集成提供了更大的灵活性。

## 连接 Provider

要将 Keep 连接到 PagerDuty：

- **Routing Key**：用于通过 PagerDuty Events API 发布事件。在 PagerDuty UI 中，这显示为集成密钥。
- **API Key**：用于通过 PagerDuty Incidents API 创建和管理事件。
- **Service Id**（可选）：如果提供，keep 将在服务范围内操作。
- **OAuth2**：令牌管理由 Keep 自动处理。

<Note>
您可以在 PagerDuty Web 应用的 **Services** > **Service Directory** > **Your service** > **Integrations** > **Expand Events API** 下找到您的路由密钥（PagerDuty UI 中的集成密钥），并选择您要使用的集成。
您可以在 PagerDuty Web 应用的 **Configuration** > **API Access** 下找到您的 API 密钥。

routing_key 用于使用 events API 向 PagerDuty 发布事件。
api_key 用于使用 incidents API 创建事件。

</Note>

### 在开源版本中启用 OAuth

如果您想在自托管 Keep 的开源版本中使用 OAuth，可以按照以下步骤操作：

1. 创建一个 PagerDuty 账户
2. 在账户页面中，转到 **Integrations** > **App Registration**
   <Frame>
     <img src="/images/pagerduty-app-registration.png" />
   </Frame>
3. 点击右上角的 **New App** 蓝色按钮
4. 填写必填字段
5. 在 Functionality 部分选择"OAuth 2.0"并点击 **Next**
6. 在 Redirect URL 中，您需要添加 Keep 的 PagerDuty OAuth2 重定向 URL，这取决于您的部署 URL。例如，如果 Keep 部署在 http://localhost:3000，则重定向 URL 为 http://localhost:3000/providers/oauth2/pagerduty
   <Frame>
     <img src="/images/pagerduty-redirect-url.png" />
   </Frame>
7. 在 Authorization 部分，选择 **Scoped OAuth** 并选择以下范围：

- Abilities: Read Access
- Incidents: Read/Write Access
- Services: Read/Write Access
- Webhook Subscriptions: Read/Write Access

8. 点击右下角的 **Register App** 蓝色按钮
9. 从 OAuth 2.0 Client Information 模态框中复制 **Client ID** 和 **Client Secret**，并在 Keep 后端部署中设置 `PAGERDUTY_CLIENT_ID` 和 `PAGERDUTY_CLIENT_SECRET` 环境变量。
   <Frame>
     <img src="/images/pagerduty-oauth2-credentials.png" />
   </Frame>

## 注意事项

provider 使用 events API 或 incidents API 来创建告警或事件。选择使用哪个 API 取决于存在 routing_key 还是 api_key。

使用免费版 PagerDuty 时，过期的试用版可能会在 Keep 中导致"pagerduty scopes are invalid"错误。

## Webhook 集成修改

webhook 集成将 Keep 添加为 Pagerduty 内"Integrations" API 中的目标。
这授予 Keep 对 Pagerduty 内以下范围的访问权限：

- `webhook_subscriptions_read`
- `webhook_subscriptions_write`
