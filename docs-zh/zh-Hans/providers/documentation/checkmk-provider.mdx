## 概述

Checkmk provider 支持 Keep 和 Checkmk 之间的无缝集成。它允许您通过 webhooks 将 Checkmk 中的告警获取到 Keep，让您可以在一个地方更轻松地管理基础设施和应用程序。

## 将 Checkmk 连接到 Keep

要将 Checkmk 连接到 Keep，您需要将其配置为 Checkmk 中的 webhook。请按照以下步骤设置集成：

1. 需要在 Checkmk 服务器上安装 Keep webhook 脚本。

2. 您可以使用以下命令下载 Keep webhook 脚本：

```bash
wget -O webhook-keep.py https://github.com/keephq/keep/blob/main/keep/providers/checkmk_provider/webhook-keep.py?raw=true
```

3. 将下载的脚本复制到 Checkmk 服务器上的以下路径：

如果您使用的是 Checkmk Docker 容器，请根据您的 docker 卷映射将其复制到以下路径：

```bash
cp webhook-keep.py /omd/sites/<site_name>/local/share/check_mk/notifications/webhook-keep.py
cd /omd/sites/<site_name>/local/share/check_mk/notifications
```

如果您使用的是安装在服务器上的 Checkmk，请将其复制到以下路径：

```bash
cp webhook-keep.py ~/local/share/check_mk/notifications/webhook-keep.py
cd ~/local/share/check_mk/notifications
```

4. 使脚本可执行：

```bash
chmod +x webhook-keep.py
```

5. 现在进入 Checkmk Web 界面并导航到 Setup。

8. 根据您的需求配置规则属性、联系人选择和条件。

9. 第一个参数是 Keep 的 Webhook URL，即 `https://api.keephq.dev/alerts/event/checkmk`。

10. 第二个参数是 Keep 的 API 密钥，您可以在 [Keep 设置](https://platform.keephq.dev/settings?selectedTab=users&userSubTab=api-keys) 中生成。

11. 点击 Save 保存配置。

12. 现在，当满足配置的条件时，您将开始通过 webhooks 接收来自 Checkmk 到 Keep 的告警。
