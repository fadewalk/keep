## 连接 Provider

<Note>
  要连接自托管 Sentry，您需要设置 `api_url` 参数。
  默认值为 `https://sentry.io/api/0/`。
</Note>

### API 密钥

要获取 Sentry API 密钥，请按照以下步骤操作（[文档](https://docs.sentry.io/product/integrations/integration-platform/?original_referrer=https%3A%2F%2Fwww.google.com%2F#internal-integrations)）：

1. 登录您的 Sentry 账户。
2. 导航 `Settings` -> `Developer Settings` 部分。
3. 点击 `Custom integrations`。
4. 点击屏幕右上角的 `Create New Integration`。

<Frame>
  <img src="/images/sentry-create-integration.png" />
</Frame>

5. 选择 `Internal Integration` 并点击 `Next`

<Frame>
  <img src="/images/sentry-internal-integration.png" />
</Frame>

6. 为集成提供一个指示性名称，例如 `Keep Integration`
7. 从权限部分，选择所需的范围：

Project: Read & Write
Issue & Event: Read
Organization: Read
Alerts: Read & Write (Not Mandatory)

<Frame>
  <img src="/images/sentry-indicative-name.png" />
</Frame>

8. 点击 `Save Changes`

<Frame>
  <img src="/images/sentry-save-changes.png" />
</Frame>

9. 向下滚动至屏幕底部的 `TOKENS` 部分，复制生成的令牌 -- 这是您将在 Keep 中使用的 API 密钥。

<Frame>
  <img src="/images/sentry-token.png" />
</Frame>

### 组织 Slug

您可以在 Sentry URL 中找到组织 Slug。
例如，这是我们的 playground 账户：`https://keep-dr.sentry.io/` - 组织 slug 是 `keep-dr`。

要从设置页面获取组织 Slug：

1. 登录您的 Sentry 账户。
2. 导航 `Settings` -> `General Settings`。
3. 从组织 Slug 输入中复制组织 Slug。

## 注意事项

<Note>
安装 Sentry webhook 集成时，Keep 会启用对所有可访问项目的内置 Webhook 集成，并添加一个新的告警，该告警具有通过 Webhooks 向所有可访问项目发送通知的`操作`。

您可以使用`内部集成`实现从 Sentry 到 Keep 的告警推送，这不是通过平台自动化的。[联系我们](mailto:founder@keephq.dev) 进行设置。

</Note>
