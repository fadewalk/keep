## 连接 Provider
1. 确保您拥有具有必要 [权限](https://docs.appdynamics.com/accounts/en/cisco-appdynamics-on-premises-user-management/roles-and-permissions) 的 AppDynamics 账户。所需的基本权限是 `Account Owner` 或 `Administrator`。或者您可以按照 [说明](https://docs.appdynamics.com/accounts/en/global-account-administration/access-management/manage-user-accounts) 创建一个账户。

## Provider 配置

1. 在 [此处](https://accounts.appdynamics.com/overview) 找到您的账户名称。
2. 获取您要安装 webhook 的 AppDynamics 实例的 appId。
3. 在 [此处](https://accounts.appdynamics.com/overview) 确定主机。

### Basic Auth 身份验证

1. 获取 AppDynamics **用户名**和**密码**。
2. 进入**身份验证**部分下的 **Basic Auth** 选项卡。
3. 输入**用户名**和**密码**。

</Frame>

### Access Token 身份验证

1. 以**账户所有者**或其他具有**管理用户、组、角色**权限的角色登录 **Controller UI**。
2. 进入**管理**。

  <img src="/images/appdynamics_1.png" width="1000" alt="AppDynamics Administration"/>

3. 进入 **API Client** 选项卡。

4. 点击 **+ Create**。

  <img src="/images/appdynamics_3.png" width="1000" alt="Create new AppDynamics API Client"/>

5. 填写客户端**名称**和**描述**。
6. 点击 **Generate Secret**。

  <img src="/images/appdynamics_4.png" width="1000" alt="AppDynamics generate API Client Secret"/>

<Tip>

7. 添加 **Account Owner** 和/或 **Administrator** 角色。

<Frame>
  <img src="/images/appdynamics_5.png" width="1000" alt="AppDynamics add API Client roles"/>

  <img src="/images/appdynamics_10.png" width="1000" alt="Keep add AppDynamics Access Token"/>

13. 输入 Access Token。

## 连接 provider

1. 确保**安装 webhook** 已勾选。
2. 点击**连接**。

## Webhook 集成修改

Webhook 集成将 Keep 添加为 AppDynamics 实例内的告警监控器。它可以在 "Alerts & Respond" 部分找到。
该集成自动获取对 AppDynamics 内以下范围的访问权限：
- `administrator`
- `authenticated`
