## 概述

**Graylog Provider** 便于通过设置 Webhook 连接从 Graylog 接收告警。它允许与 Graylog 无缝集成，通过 Keep 接收有关事件和告警的通知。

<AutoGeneratedSnippet />

## 连接 Provider

1. 通过遵循 [Graylog 的 API 访问文档](https://go2docs.graylog.org/current/setting_up_graylog/rest_api_access_tokens.htm?tocpath=Set%20up%20Graylog%7CGet%20Started%20with%20Graylog%7CREST%C2%A0API%7C_____3#CreateanAccessToken) 从您的 Graylog 实例获取**用户名**和**访问令牌**。
2. 将**部署 URL** 设置为您的 Graylog 实例的基础 URL（例如 `http://127.0.0.1:9000`）。
3. 确保用户在 Graylog 中具有**Admin** 角色。

## 功能

**Graylog Provider** 支持以下主要功能：

- **Webhook 设置**：配置 webhooks 以将告警发送到 Keep。
- **告警获取**：根据指定的搜索参数从 Graylog 获取和格式化告警（仅最多 10000 条最近的告警）。

<Note>
确保 `page` 和 `per_page` 的乘积不超过 10,000。
</Note>

<Note>
Graylog v4.x 的通知 URL 将 api_key 作为查询参数，这是默认行为。
</Note>
