## 认证
此提供商需要身份验证。
- **token**: Token（必需：True，敏感：True）
- **host**: Grafana 主机（必需：True，敏感：False）
- **datasource_uid**: 数据源 UID（必需：False，敏感：False）

某些范围可能需要才能通过提供商执行特定操作或查询。以下是相关范围及其用例的摘要：
- **alert.rules:read**: 在文件夹及其子文件夹中读取 Grafana 警报规则。（强制）（[文档](https://grafana.com/docs/grafana/latest/administration/roles-and-permissions/access-control/custom-role-actions-scopes/)）
- **alert.provisioning:read**: 通过配置 API 读取所有 Grafana 警报规则、通知策略等。([文档](https://grafana.com/docs/grafana/latest/administration/roles-and-permissions/access-control/custom-role-actions-scopes/)）
- **alert.provisioning:write**: 通过配置 API 更新所有 Grafana 警报规则、通知策略等。([文档](https://grafana.com/docs/grafana/latest/administration/roles-and-permissions/access-control/custom-role-actions-scopes/)）

## 通过 Webhook 连接（双向）
此提供商支持 webhook。

如果您的 Grafana 无法从 Keep 访问，您可以使用以下 webhook URL 配置 Grafana 将警报发送到 Keep：

    1. 在 Grafana 中，转到 Grafana 仪表板中的"警报"选项卡。
    2. 点击左侧边栏中的"联系人"并创建一个新的。
    3. 为其命名并选择 Webhook 作为联系人类型，webhook URL 为 KEEP_BACKEND_URL/alerts/event/grafana。
    4. 添加 'X-API-KEY' 作为请求头 {api_key}。
    5. 保存 webhook。
    6. 点击左侧边栏中的"通知策略"
    7. 在"默认策略"下点击"新建子策略"
    8. 删除所有匹配器，直到看到以下内容："如果未指定匹配器，此通知策略将处理所有警报实例。"
    9. 选择您刚在联系人下创建的 webhook 联系人，然后点击"保存策略"
