## 认证
此提供商需要身份验证。
- **VMAlertHost**: VMAlert 运行的主机名或 IP 地址（必需：False，敏感：False）
- **VMAlertPort**: VMAlert 正在监听的端口号（必需：False，敏感：False）
- **VMAlertURL**: VMAlert 实例的完整 URL。Host/Port 的替代方案（必需：False，敏感：False）
- **VMBackendHost**: VictoriaMetrics 后端运行的主机名或 IP 地址（必需：False，敏感：False）
- **VMBackendPort**: VictoriaMetrics 后端正在监听的端口号（必需：False，敏感：False）
- **VMBackendURL**: VictoriaMetrics 后端的完整 URL。Host/Port 的替代方案（必需：False，敏感：False）
- **BasicAuthUsername**: 基本身份验证的用户名（必需：False，敏感：False）
- **BasicAuthPassword**: 基本身份验证的密码（必需：False，敏感：True）
- **SkipValidation**: 输入 'true' 跳过身份验证（必需：False，敏感：False）
- **insecure**: 跳过 TLS 验证（必需：False，敏感：False）

## 通过 Webhook 连接（双向）

此提供商利用 Prometheus Alertmanager 的可配置 webhook。使用以下模板配置 AlertManager：

```
route:
  receiver: "keep"
  group_by: ['alertname']
  group_wait:      15s
  group_interval:  15s
  repeat_interval: 1m
  continue: true

receivers:
- name: "keep"
  webhook_configs:
  - url: 'KEEP_BACKEND_URL/alerts/event/victoriametrics'
    send_resolved: true
    http_config:
      basic_auth:
        username: api_key
        password: {api_key}

```
