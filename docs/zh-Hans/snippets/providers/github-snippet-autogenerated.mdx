## 在工作流中

作为"步骤"来查询数据，示例：
```yaml
steps:
    - name: Query github
      provider: github
      config: "{{ provider.my_provider_name }}"
      with:
        repository: {value}
        previous_stars_count: {value}
        last_stargazer: {value}
```


作为"操作"来进行更改或更新数据，示例：
```yaml
actions:
    - name: Query github
      provider: github
      config: "{{ provider.my_provider_name }}"
      with:

        repo_owner: {value}  # 仓库所有者。
        ref: {value}  # 要使用的 ref。
        inputs: {value}  # 要使用的输入。
```

## 提供商方法

- **get_last_commits** 获取 GitHub 仓库中最近 N 次提交（视图，范围：无附加范围）

    - `repository`: 要获取提交的 GitHub 仓库。
    - `n`: 要获取的提交数量。
- **get_last_releases** 获取 GitHub 仓库中最近 N 个版本及其变更日志（视图，范围：无附加范围）

    - `repository`: 要获取版本的 GitHub 仓库。
    - `n`: 要获取的版本数量。
