## 在工作流中

此提供商可用于工作流。


作为"步骤"来查询数据，示例：
```yaml
steps:
    - name: Query keep
      provider: keep
      config: "{{ provider.my_provider_name }}"
      with:
        filters: {value}  # 过滤查询 Keep（仅适用于版本 1）
        version: {value}  # Keep API 版本
        distinct: {value}  # 如果为 True，仅返回不同的警报
        time_delta: {value}  # 查询 Keep 的时间增量（天）
        timerange: {value}  # 计算时间增量的时间范围字典
        filter: {value}  # 过滤查询 Keep（仅适用于版本 2）
        limit: {value}  # 限制结果数量（仅适用于版本 2）
```


作为"操作"来进行更改或更新数据，示例：
```yaml
actions:
    - name: Query keep
      provider: keep
      config: "{{ provider.my_provider_name }}"
      with:
        delete_all_other_workflows: {value}  # 如果为 True，删除所有其他工作流
        workflow_full_sync: {value}  # 如果为 True，同步所有工作流
        workflow_to_update_yaml: {value}  # 要更新的工作流 yaml
        alert: {value}  # 要创建的警报数据
        fingerprint_fields: {value}  # 用于警报指纹的字段
        override_source_with: {value}  # 覆盖警报来源
        read_only: {value}  # 如果为 True，不修改现有警报
        fingerprint: {value}  # 警报指纹
        if: {value}  # 评估警报创建的条件
        for: {value}  # 状态警报的持续时间
```
