## 认证
此提供商需要身份验证。
- **routing_key**: 路由密钥（集成或规则集密钥）（必需：False，敏感：False）
- **api_key**: API 密钥（用户或团队 API 密钥）（必需：False，敏感：True）
- **oauth_data**: 用于 OAuth 流程（必需：False，敏感：True）
- **service_id**: 服务 ID（如果提供，Keep 将仅在此服务上操作）（必需：False，敏感：False）

某些范围可能需要才能通过提供商执行特定操作或查询。以下是相关范围及其用例的摘要：
- **incidents_read**: 读取事件数据。（强制）
- **incidents_write**: 写入事件。
- **webhook_subscriptions_read**: 读取 webhook 数据。
- **webhook_subscriptions_write**: 写入 webhooks。



## 在工作流中

此提供商可用于工作流。


作为"步骤"来查询数据，示例：
```yaml
steps:
    - name: Query pagerduty
      provider: pagerduty
      config: "{{ provider.my_provider_name }}"
      with:
        incident_id: {value}
        incident_key: {value}
```


作为"操作"来进行更改或更新数据，示例：
```yaml
actions:
    - name: Query pagerduty
      provider: pagerduty
      config: "{{ provider.my_provider_name }}"
      with:
        title: {value}  # 警报或事件的标题
        dedup: {value}  # 用于事件 API 去重的字符串，最多 255 个字符
        service_id: {value}  # 事件的服务 ID
        routing_key: {value}  # API routing_key（可选），如果未指定，则回退到提供商中提供的那个
        requester: {value}  # 请求创建事件的用户电子邮件
        incident_id: {value}  # 识别事件的密钥。如果未提供则生成 UUID
        event_type: {value}  # 事件 API 的事件类型（trigger/acknowledge/resolve）
        severity: {value}  # 事件 API 的严重程度（critical/error/warning/info）
        source: {value}  # 事件 API 的来源字段
        priority: {value}  # 事件优先级参考 ID
        status: {value}  # 事件更新的状态（resolved/acknowledged）
        resolution: {value}  # 已解决事件的解决备注
        body: {value}  # 事件的正文，详见 https://developer.pagerduty.com/api-reference/a7d81b0e9200f-create-an-incident#request-body
        kwargs: {value}  # 附加的事件/事件字段
```
