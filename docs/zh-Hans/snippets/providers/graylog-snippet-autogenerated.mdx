## 认证
此提供商需要身份验证。
- **graylog_user_name**: 用户名（必需：True，敏感：False）
- **graylog_access_token**: Graylog 访问令牌（必需：True，敏感：True）
- **deployment_url**: 部署 URL（必需：True，敏感：False）
- **verify**: 验证 SSL 证书（必需：False，敏感：False）

某些范围可能需要才能通过提供商执行特定操作或查询。以下是相关范围及其用例的摘要：
- **authenticated**: 所有操作的必需项，确保用户已身份验证。（强制）
- **authorized**: 查询事件和管理资源的必需项，确保用户具有 `Admin` 权限。（强制）

## 通过 Webhook 连接（双向）
此提供商支持 webhook。


要将警报从 Graylog 发送到 Keep，请使用以下 webhook URL 配置 Graylog 将警报发送到 Keep：

1. 在 Graylog 中，从顶部栏转到 `警报` > `通知`。
2. 点击"创建通知"。
3. 在新通知表单中，配置：

**注意**：对于 Graylog v4.x，请将 **URL** 设置为 `KEEP_BACKEND_URL/alerts/event/graylog?api_key={api_key}`。

- **显示名称**: keep-graylog-webhook-integration
- **标题**: keep-graylog-webhook-integration
- **通知类型**: 自定义 HTTP 通知
- **URL**: KEEP_BACKEND_URL/alerts/event/graylog  # 将此 URL 加入白名单
- **头**: X-API-KEY:{api_key}
4. 擦除正文模板。
5. 点击"创建通知"。
6. 转到`事件定义`选项卡，选择将触发您要发送到 Keep 的警报的事件定义，然后点击更多 > 编辑。
7. 转到"通知"选项卡。
8. 点击"添加通知"并选择您在步骤 3 中创建的"keep-graylog-webhook-integration"。
9. 点击"添加通知"。
10. 点击`下一步` > `更新`事件定义
