title: "对比"
---

通过将工具与同一生态系统中的其他工具进行比较，往往更容易理解工具的功能。在这里，我们将解释 Keep 如何与这些工具交互和比较。

## Keep 与 IRM（PagerDuty、OpsGenie 等）的对比

事件管理工具旨在在正确的时间通知正确的人、简化报告并建立高效的战情室。

"Keep"专注于告警生命周期、噪音减少和 AI 驱动的事件关联。本质上，Keep 作为"IRM 之前的智能层"，在数百万告警到达 IRM 工具之前对其进行管理。Keep 提供与 PagerDuty、OpsGenie、Grafana OnCall 等的高质量集成。

## Keep 与可观测性中的 AIOps（Elastic、Splunk 等）的对比

Keep 与众不同，因为它能够在不同的可观测性平台之间关联告警。

|                                       | Keep                                                          | 替代方案                      |
| ------------------------------------- | -------------------------------------------------------------- | ---------------------------- |
| 从单一平台聚合告警 | ✅                                                           | ✅                            |
| 从多个平台聚合告警 | ✅                                                             | ❌                            |
| 在多个来源之间关联告警 | ✅                                                    | ❌                            |
| 告警富集                     | ✅                                                             | ❌                            |
| 开源                           | ✅                                                             | ❌                            |
| 工作流自动化                   | ✅                                                             | ❌                            |

## Keep 与 AIOps 平台（BigPanda、Moogsoft 等）的对比

Keep 是 BigPanda 和 Moogsoft 等平台的替代方案。
使用过传统平台和 Keep 的客户注意到告警关联方面有显著改善。与其他平台的手动方法不同，Keep 使用先进的最新 AI 模型来进行更简单、更有效的告警关联。

|                                       | Keep                                                          | 替代方案                      |
| ------------------------------------- | -------------------------------------------------------------- | ---------------------------- |
| 告警聚合                 | ✅                                                             | ✅                            |
| 集成                          | ✅ (双向)                                            | ✅ (Webhook)                 |
| 告警富集                     | ✅                                                             | ✅                            |
| 开源                           | ✅                                                             | ❌                            |
| 工作流自动化                   | ✅ (类似 GitHub Actions，基础设施即代码)               | ✅                            |
| 托管版本                       | ✅                                                             | ✅                             |
| 本地部署                           | ✅                                                             | ❌                             |
| 噪音减少与关联         | ✅ (AI)                                                        | ✅ (某些情况下基于规则) |
