## 什么是 AIOps？

简而言之，用于 IT 运营的 AI（AIOps）就是自动化重复性任务、减少监控工具的噪音，并通过将压倒性的数据转化为可操作的洞察来帮助团队克服告警疲劳。

借助 AIOps，团队可以消除噪音、优先处理关键问题，并专注于解决真正的问题，而不是不断地扑灭告警。

## 我们为什么要构建 Keep？

在使用 BigPanda、Splunk ITSI 或 ServiceNow ITOM 等当前工具时，我们发现了一个差距：

- **没有开源解决方案：** 我们有用于可视化的 Grafana 和用于指标的 Prometheus，但没有用于 AIOps 的解决方案。Keep 作为第一个开源 AIOps 解决方案填补了这一空白。
- **对 DevOps/SRE 不友好：** 当前工具是企业级的，但并不友好。如果您是 ~100 名员工公司的 SRE 团队负责人或 IT 运营负责人，现有工具对您不起作用。它们太贵了，而且 UX 需要专用团队来进行设置和维护。Keep 是企业级的（可扩展、SSO 等），但也为想要采用 AIOps 实践的小型团队设计。
- **"后 LLM 时代"的 AIOps：** 现有工具构建于不同的技术时代。Keep 旨在利用大型语言模型（LLM）时代的进步，将 AI 更无缝地集成到 IT 运营中。

## 我们的理念

- **易于开始** - 无论是在本地还是 Kubernetes 上，我们都提供一键解决方案，如 `helm install` 和 `docker-compose`，以便您可以快速启动 Keep 并开始探索其功能。
- **易于扩展** - Keep 在设计时考虑到了可扩展性，使添加新集成或功能以满足您的特定需求变得简单直接。
- **易于部署** - Keep 的每个方面都可以作为代码进行配置，实现部署的无缝自动化并集成到您的 CI/CD 管道中。
- **易于协作** - 作为一个开源项目，我们真正相信社区和协作的力量。我们积极倾听用户反馈，并根据用户的需求和见解，努力不断改进 Keep。

## 我们的愿景

构建 Keep 的目的是让每个团队都能从 AIOps 中受益。

无论您是一个寻找 Kubernetes 本地 Prometheus 告警单窗格玻璃的小型团队，还是拥有数十个生成告警的工具并需要与 ServiceNow 工单同步的企业，Keep 都适合您。

我们的愿景是民主化 AIOps，让不同规模的团队都能使用并受益于 AIOps。
