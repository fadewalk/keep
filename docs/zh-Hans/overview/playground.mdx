## 告警

通过可自定义的预设获得所有告警的单一窗口视图。使用 CEL（通用表达式语言）语法进行精确过滤，配置告警表布局以匹配您的工作流程，并探索分面以快速洞察告警模式和指标。

## 事件

详细检查事件，包括其关联的告警和时间线。测试将相关告警分组为事件的关联逻辑和映射配置，并验证您的抑制或解决策略。

## 提供者

与 Prometheus、Datadog 或 GCP Monitoring 等外部数据源或告警提供者集成。配置和测试映射以确保从各种来源正确摄取和规范化数据到 Keep 的统一架构中。

## 工作流

精确构建和测试自动化工作流来管理告警和事件。同时使用直观的 UI 构建器和高级脚本功能进行实验，以基于动态条件触发操作、通知或外部集成。

## AIOps 技术

测试和完善去重、丰富、映射和提取规则以优化告警处理。尝试这些技术将原始告警转换为可操作的数据并有效减少噪声。
