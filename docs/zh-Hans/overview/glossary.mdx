## 告警
告警是在发生坏事或即将发生时触发的事件。
术语"告警"有时可以与"警报"（例如 CloudWatch 中的）或"监控器"（Datadog 中的）互换使用。

## 事件
事件是一组相互关联的告警。

## 提供者
提供者可以是模块，用于将告警拉入 Keep 或通过与外部系统交互将数据推出 Keep。

### 作为数据源的提供者
在工作流的上下文中，提供者可以：
- 查询数据 - 查询 Datadog 的 API 或对数据库运行 SQL 查询。
- 推送数据 - 发送 Slack 消息或创建 PagerDuty 事件。

### 作为告警源的提供者
当您连接提供者时，Keep 开始从该提供者读取和处理告警。例如，连接 Prometheus 实例后，您将在 Keep 中开始看到 Prometheus 告警。
提供者可以向 Keep 推送告警，或者 Keep 可以从提供者拉取告警。

#### 向 Keep 推送告警（手动）
您可以配置告警源将告警推送到 Keep。

例如，考虑 Prometheus。如果要将告警从 Prometheus 推送到 Keep，您需要配置 Prometheus Alertmanager 使用 API 密钥认证向
'https://api.keephq.dev/alerts/event/prometheus' 发送告警。每个提供者都实现了推送机制，并在特定提供者页面下有文档说明。

#### 向 Keep 推送告警（自动）
在兼容工具中，Keep 可以自动与源工具的告警策略集成，并添加自身作为告警目的地。您可以在[此处](/providers/overview)了解有关 Webhook 集成的更多信息。
请注意，这会稍微修改您的监控器/通知策略。

### Keep 拉取告警
Keep 还与各种工具的告警 API 集成，可以自动拉取告警。虽然拉取更容易设置（只需要凭证），但当涉及自动化时，推送更可取。

## 工作流
工作流由一系列[步骤](/workflows/overview#steps)和[动作](/workflows/overview#actions)组成。
工作流可以通过以下方式触发：
- 触发告警时。
- 预定义的时间间隔。
- 手动。

工作流通常用于：
1. 用更多上下文富集您的告警。
2. 自动化对告警的响应。
3. 创建多步告警。

## API 优先
Keep 是一个 API 优先平台，这意味着您通过 UI 执行的任何操作也可以通过 [API](https://api.keephq.dev/redoc) 完成
这为您提供了将 Keep 与现有技术栈集成以及自动化告警修复和富集过程的灵活性。
