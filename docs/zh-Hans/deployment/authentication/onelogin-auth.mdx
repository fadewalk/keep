本文档提供了关于 Keep 中 OneLogin 集成的综合信息。

## 概述

Keep 支持将 OneLogin 作为认证提供商，实现：
- 通过 OneLogin 实现单一登录 (SSO)
- OAuth2/OIDC 认证流程
- 令牌刷新功能
- 通过自定义声明进行基于角色的访问控制
- 通过 NextAuth.js 进行会话管理

## 环境变量

### 后端环境变量

| 变量 | 描述 | 示例 |
|----------|-------------|---------|
| `AUTH_TYPE` | 设置为 `"ONELOGIN"` 以启用 OneLogin 认证 | `ONELOGIN` |
| `ONELOGIN_ISSUER` | 您的 OneLogin 应用程序的发行者 URL | `https://company.onelogin.com/oidc/2` |
| `ONELOGIN_CLIENT_ID` | 您的 OneLogin 应用程序的客户端 ID | `abc123def456ghi789` |
| `ONELOGIN_CLIENT_SECRET` | 您的 OneLogin 应用程序的客户端密钥 | `abcd1234efgh5678ijkl9012` |
| `ONELOGIN_ADMIN_ROLE` | 映射到 keep 管理员角色的角色 | `KeepAdmin` |
| `ONELOGIN_NOC_ROLE` | 映射到 keep noc 角色的角色 | `KeepNoc` |
| `ONELOGIN_WEBHOOK_ROLE` | 映射到 keep webhook 角色的角色 | `KeepWebhook` |
| `ONELOGIN_AUTO_CREATE_USER` | 是否尝试在 keep 中自动创建用户 | `True` |

## OneLogin 配置

### 创建 OneLogin 应用程序

1. 登录到您的 OneLogin 管理控制台
2. 导航到 **应用程序**
3. 点击 **添加应用**
4. 搜索 **OpenId Connect (OIDC)** 并选择它
5. 点击 **保存**

### 应用程序设置

1. **显示名称**：输入您的应用程序名称（例如"Keep"）
2. **重定向 URI**：输入您应用的回调 URL，例如 `https://your-keep-domain.com/api/auth/callback/onelogin`
3. **登录 URL**：输入您应用的登录 URL，例如 `https://your-keep-domain.com/signin`
4. **角色映射**：
    - 转到 Parameters 选项卡
    - 将组映射到用户角色或组，默认值为分号分隔的输入值
5. 转到 **SSO** 选项卡并配置：
   - **应用程序类型**：Web
   - **令牌端点**：客户端密钥发布
6. **访问**：
   - 分配给适当的角色或用户
7. 点击 **保存**
8. 从 SSO 选项卡复制客户端 id、客户端密钥和发行者 URL
