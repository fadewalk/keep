---
title: "无认证"
---
<Warning>在生产环境中使用此配置不安全，强烈建议不要这样做。</Warning>


部署 Keep 而不使用认证是最快的启动方式，非常适合本地开发或安全不是问题的内部工具。
## 设置说明
无论您使用 docker-compose、kubernetes、openshift 还是任何其他部署方法，请添加以下环境变量：
```
# Frontend
AUTH_TYPE=NOAUTH

# Backend
AUTH_TYPE=NOAUTH
```
## 影响
使用 `AUTH_TYPE=NOAUTH`：
- Keep 不会显示任何登录页面，并将允许您在不进行认证的情况下使用 API。
- Keep 将使用"keep"作为租户 ID 的 JWT，但不会验证它。
- `x-api-key` 头中提供的任何 API 密钥都将被接受而不进行验证。

此配置基本上绕过了所有认证检查，使其不适合安全是问题的生产环境。
