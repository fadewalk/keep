## 背景

Keep 可以通过环境变量高度配置。这允许您自定义后端和前端组件的各个方面，而无需修改代码。环境变量可以在您的部署环境中设置，例如在 Kubernetes 配置、Docker Compose 文件中或直接在主机系统上设置。

## 后端环境变量

### 常规

<Info>
  常规配置变量控制 Keep 服务器的核心行为。
  这些设置决定基本方面，例如服务器的主机、端口，
  以及是否启用某些组件（如调度器和消费者）。
</Info>

### 服务网格（内部告警摄取）

<Info>
  这些设置允许同一 Kubernetes 集群内的受信任服务在
  不需要 Keep API 密钥的情况下向 Keep POST 告警。这适用于
  服务到服务的通信，其中网络级认证（例如
  Istio mTLS 与 AuthorizationPolicy）确保只有授权的调用者可以
  访问 Keep 的告警摄取端点。
</Info>

|                   Env var                    |                                   用途                                    | 必填 | 默认值 |    有效选项     |
| :------------------------------------------: | :--------------------------------------------------------------------------: | :--: | :----: | :------------: |
| **KEEP_ALLOW_MESH_ALERT_INGESTION**          | 允许对 `/alerts/event*` 端点进行未认证的 POST 请求           |   否   |  "false"   | "true" 或 "false"   |

当 `KEEP_ALLOW_MESH_ALERT_INGESTION` 设置为 `"true"` 时，不携带 API 密钥或 bearer 令牌的 `/alerts/event*` 请求将被接受，并作为具有 `webhook` 角色的内部服务进行认证。

调用服务可以选择设置 `X-Service-Name` HTTP 头，以在 Keep 的日志和审计跟踪中标识自己：

```bash
curl -X POST http://keep-backend:8080/alerts/event \
  -H "Content-Type: application/json" \
  -H "X-Service-Name: my-service" \
  -d '[{"id":"alert-1","name":"Example Alert","severity":"info","status":"firing","source":["my-service"]}]'
```

认证实体将具有：
- **email**：`service:<X-Service-Name 头值>`（如果未设置头，则默认为 `service:unknown`）
- **role**：`webhook`（授予 `write:alert` 和 `write:incident` 范围）

<Warning>
  此功能绕过告警摄取端点的 API 密钥认证。您**必须**将其与网络级访问控制（如
  Istio AuthorizationPolicy）配对，以限制哪些服务可以访问这些
  端点。如果没有网络级 enforcement，任何可以访问
  Keep 后端的客户端都可以 POST 告警。
</Warning>

### 速率限制

<Info>
  速率限制配置控制可以在指定时间范围内向 Keep's
  API 端点发出的请求数量。这有助于防止滥用并
  确保系统稳定性。
</Info>

|          Env var           |                用途                | 必填 |   默认值    |                                     有效选项                                     |
| :------------------------: | :-----------------------------------: | :--: | :--------: | :-------------------------------------------------------------------------------: |
|    **KEEP_USE_LIMITER**    |   启用或禁用速率限制   |   否   |  "false"   |                                   "true" 或 "false"                                   |
| **KEEP_LIMIT_CONCURRENCY** | 设置 API 端点的速率限制 |   否   | "100/minute"  | 格式："{number}/{interval}"，其中 interval 可以是 "second"、"minute"、"hour"、"day" |

<Note>
目前，速率限制适用于以下端点：
- POST `/alerts/event` - 通用事件摄取端点
- POST `/alerts/{provider_type}` - 提供商特定的事件摄取端点

当启用 `KEEP_USE_LIMITER` 时，这些端点根据 `KEEP_LIMIT_CONCURRENCY` 设置进行速率限制。

</Note>
