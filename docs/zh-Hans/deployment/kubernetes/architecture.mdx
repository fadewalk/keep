## 高级架构
Keep 架构由两个主要组件组成：

1. **Keep API** - 基于 FastAPI 的后端服务器，处理业务逻辑和 API 端点。
2. **Keep Frontend** - 基于 Next.js 的前端界面，用于用户交互。
3. **Websocket 服务器** - 用于实时更新而不刷新页面的 Soketi 服务器。
4. **数据库服务器** - 用于存储和管理持久化数据的数据库。支持 SQLite、PostgreSQL、MySQL 和 SQL Server。

## Kubernetes 架构

Keep 使用单个统一的 NGINX ingress 控制器将流量路由到所有组件（frontend、backend 和 websocket）。ingress 处理基于路径的路由：

默认情况下：
- `/` 路由到 **Frontend**（可通过 `global.ingress.frontendPrefix` 配置）
- `/v2` 路由到 **Backend**（可通过 `global.ingress.backendPrefix` 配置）
- `/websocket` 路由到 **WebSocket**（可通过 `global.ingress.websocketPrefix` 配置）

### Kubernetes 配置
<Tip>本节仅涵盖 kubernetes 特定的配置。要了解 Keep 特定的配置（由环境变量控制），请参阅 [Keep 配置](/deployment/configuration)</Tip>

这些组件中的每一个都可以通过 Helm chart 中的 `values.yaml` 文件进行自定义。


以下是每个组件可以调整的关键配置。

#### 1. Frontend 配置
```yaml
frontend:
  enabled: true                 # 启用或禁用 frontend deployment。
  replicaCount: 1               # Frontend 副本数量。
  image:
    repository: us-central1-docker.pkg.dev/keephq/keep/keep-ui
    pullPolicy: Always          # 镜像拉取策略（Always、IfNotPresent）。
    tag: latest
  serviceAccount:
    create: true                # 创建新的服务账户。
    name: ""                    # 服务账户名称（空为默认）。
  podAnnotations: {}            # Frontend pod 的注解。
  podSecurityContext: {}        # Frontend pod 的安全上下文。
  securityContext: {}           # 容器的安全上下文。
  service:
    type: ClusterIP              # 服务类型（ClusterIP、NodePort、LoadBalancer）。
    port: 3000                  # Frontend 服务暴露的端口。
```

#### 2. Backend 配置
```yaml
backend:
  enabled: true                # 启用或禁用 backend deployment。
  replicaCount: 1              # Backend 副本数量。
  image:
    repository: us-central1-docker.pkg.dev/keephq/keep/keep-api
    pullPolicy: Always         # 镜像拉取策略（Always、IfNotPresent）。
  serviceAccount:
    create: true               # 创建新的服务账户。
    name: ""                   # 服务账户名称（空为默认）。
  podAnnotations: {}           # Backend pod 的注解。
  podSecurityContext: {}       # Backend pod 的安全上下文。
  securityContext: {}          # 容器的安全上下文。
  service:
    type: ClusterIP      # 服务类型（ClusterIP、NodePort、LoadBalancer）。
    port: 8080           # Backend API 暴露的端口。
```

#### 3. WebSocket 服务器配置
Keep 使用 Soketi 作为其 websocket 服务器。要了解如何配置它，请参阅 [Soketi 文档](https://github.com/soketi/charts/tree/master/charts/soketi)。


#### 4. 数据库配置
Keep 支持大量数据库（例如 postgresql、mysql、sqlite 等）。在此描述如何将它们全部部署到 k8s 超出了范围。如果您有具体问题 - [联系我们](https://slack.keephq.dev)，我们将很乐意提供帮助。
