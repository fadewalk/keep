## 入门

要开始使用 Keep GitHub 应用程序，请按照以下简单步骤操作：

1. 注册并登录 **[Keep 平台](https://platform.keephq.dev)**。
2. 通过入门屏幕或访问 **[此链接](https://github.com/apps/keephq)** 安装 **Keep GitHub 应用程序**。安装过程简单直观。

   <Frame>
     <img src="/images/github-app-install.png" />
   </Frame>

3. 连接您首选的提供商（如 Datadog），将其链接到 Keep 平台。这一步允许 Keep 无缝生成警报并将其部署到您选择的提供商。

   <Frame>
     <img src="/images/connect-provider.png" />
   </Frame>

4. 您已准备就绪！Keep GitHub 应用程序已成功集成到您的 GitHub 工作流中。

## 工作原理

Keep GitHub 应用程序在后台无缝运行，确保您了解仓库中的相关变更。每当拉取请求打开或更新时，应用程序会监控 `.keep/` 目录下的文件。

一旦检测到变更，GitHub 应用程序会向 Keep 的 API 智能 AI 层发送 HTTP 请求。AI 层分析变更文件的内容，并结合来自提供商（现有警报、示例日志等）的上下文，根据用户提供的纯英文描述生成警报。AI 驱动的警报生成确保了准确性和相关性。

警报生成后，Keep GitHub 应用程序会自动在拉取请求中的相应文件上评论警报。这允许您作为用户方便地审查和验证生成的警报。

如果生成的警报符合您的要求并准备好部署，您可以简单地在文件上留下评论。评论应包含预定义的表情符号之一，例如 🚀 或 🆗（请参阅["使用表情符号部署警报"](#deploying-alerts-with-emojis) 部分）。Keep GitHub 应用程序将这些表情符号识别为继续部署过程的命令。

这个直观的工作流程简化了警报生成和部署过程，为您提供了无缝体验，让您能够专注于项目的核心方面。

## 监控 .keep/ 目录下的文件

Keep GitHub 应用程序积极监控位于仓库父级的 `.keep/` 目录中存在的文件。对这些文件的任何更改或更新都将触发警报生成过程。这让您能够专注于项目的基本方面，同时确保及时识别和处理相关变更。

## 警报文件结构

`.keep/` 目录下的每个文件代表一个警报。警报文件的结构遵循 YAML 格式。以下是警报文件的示例：

```yaml title=alert-example.yaml
# The alert text in plain English
alert: |
  Count the error rate (4xx-5xx) this service has in the last 10 minutes.
  Alert when the threshold is above 5% out of total requests.
  Send a Slack message to the #alerts-playground channel and include all the context you have"

# The provider you've previously connected and want this alert to be generated for
provider: datadog
# You can use this to override Keep's managed API and have the GitHub application
# use the API that you run locally (using the NGROK URL)
# api_url: https://OVERRIDE-KEEP-MANAGED-API
```

警报文件由以下组件构成：

1. **警报文本**：此部分包含警报的纯英文描述。编写清晰简洁的说明，描述应触发警报的条件或标准。您可以包括任何相关上下文以便于理解和解决。

2. **提供商**：指定要生成警报的提供商。这确保警报与您现有的监控和通知基础设施无缝集成。在上面的示例中，警报配置为针对 Datadog 生成。

3. **API 覆盖**：可选地，您可以包含 api_url 字段来覆盖 Keep 管理的 API。这允许您使用本地托管的 API 进行高级定制和集成。

## 使用表情符号部署警报

要将警报部署到指定的提供商，您可以在相应的文件上留下包含 🚀 或 🆗 表情符号的评论。Keep GitHub 应用程序将这些表情符号识别为命令，并将相应地启动部署过程。这种流线型方法确保了部署警报时的顺畅直观体验。

例如，通过留下带有 🚀 表情符号的评论，您可以向 Keep GitHub 应用程序发出信号，将警报部署到指定的提供商（以上示例中的 Datadog）。

<Frame>
  <img src="/images/first-alert.yaml.png" />
</Frame>

Keep GitHub 应用程序将用 👍 标记评论，表示警报已成功部署，或者用 👎 标记，并在情况下在另一个评论中说明失败原因。

<Info>
  Keep GitHub 应用程序有重试机制，如果警报未能成功部署到提供商，会自动尝试修复。
  如果部署的警报与最初生成的警报不同，Keep Github 应用程序将再次评论更新后的警报。
</Info>
