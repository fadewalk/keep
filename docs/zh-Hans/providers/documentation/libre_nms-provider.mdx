## 将 LibreNMS 连接到 Keep

1. 打开 LibreNMS 仪表板，点击右上角的设置。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_4.png" />
</Frame>

2. 点击 `Create API access token` 生成新的 API 密钥。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_5.png" />
</Frame>

3. 为 API 密钥提供描述，然后点击 `Create API Token`。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_6.png" />
</Frame>

## Webhooks 集成

1. 打开 LibreNMS 仪表板，打开导航栏中的 `Alerts` 选项卡，点击 `Alert Transports`。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_1.png" />
</Frame>

2. 点击 `Create add transport`，将 `Transport type` 选择为 `API`。将 `API Method` 选择为 `POST`。

3. 将 `API URL` 填写为 [https://api.keephq.dev/alerts/event/libre_nms](https://api.keephq.dev/alerts/event/libre_nms)。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_2.png" />
</Frame>

4. 复制下面的 JSON 并粘贴到 `body` 字段中。

```json
{
  "title": "{{ $title }}",
  "hostname": "{{ $hostname }}",
  "device_id": "{{ $device_id }}",
  "sysDescr": "{{ $sysDescr }}",
  "sysName": "{{ $sysName }}",
  "sysContact": "{{ $sysContact }}",
  "os": "{{ $os }}",
  "type": "{{ $type }}",
  "ip": "{{ $ip }}",
  "display": "{{ $display }}",
  "version": "{{ $version }}",
  "hardware": "{{ $hardware }}",
  "features": "{{ $features }}",
  "serial": "{{ $serial }}",
  "status": "{{ $status }}",
  "status_reason": "{{ $status_reason }}",
  "location": "{{ $location }}",
  "description": "{{ $description }}",
  "notes": "{{ $notes }}",
  "uptime": "{{ $uptime }}",
  "uptime_short": "{{ $uptime_short }}",
  "uptime_long": "{{ $uptime_long }}",
  "elapsed": "{{ $elapsed }}",
  "alerted": "{{ $alerted }}",
  "alert_id": "{{ $alert_id }}",
  "alert_notes": "{{ $alert_notes }}",
  "proc": "{{ $proc }}",
  "rule_id": "{{ $rule_id }}",
  "id": "{{ $id }}",
  "faults": "{{ $faults }}",
  "uid": "{{ $uid }}",
  "severity": "{{ $severity }}",
  "rule": "{{ $rule }}",
  "name": "{{ $name }}",
  "string": "{{ $string }}",
  "timestamp": "{{ $timestamp }}",
  "contacts": "{{ $contacts }}",
  "state": "{{ $state }}",
  "msg": "{{ $msg }}",
  "builder": "{{ $builder }}"
}
```

5. 按照以下步骤在 Keep 中创建新的 API 密钥。

6. 进入 Keep 仪表板，点击左下角的个人资料图标，点击 `Settings`。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-1.png" />
</Frame>

7. 选择 `Users and Access` 选项卡，然后选择 `API Keys` 选项卡，创建一个新的 API 密钥。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-2.png" />
</Frame>

8. 提供名称，选择角色为 `webhook`，然后点击 `Create API Key`。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-3.png" />
</Frame>

9. 复制 API 密钥。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-4.png" />
</Frame>

10. 添加一个新的 header，key 为 `X-API-KEY`，在 Keep 中创建一个新的 API 密钥，并将其作为值粘贴，然后保存 webhook。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_2.png" />
</Frame>

11. 保存 webhook。

12. 您可以从 LibreNMS 仪表板的 Devices 选项卡添加设备，并选择您创建的告警传输。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_3.png" />
</Frame>

13. 现在，您将在 Keep 中收到告警。
