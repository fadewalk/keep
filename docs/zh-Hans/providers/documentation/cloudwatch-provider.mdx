## 概述

CloudWatch Provider 提供与 AWS CloudWatch 的直接集成，使 Keep 用户能够在 Keep 平台内接收 CloudWatch 告警。此集成集中了监控和告警功能，允许对基础设施或应用程序运行状况的变化做出及时响应。

### 主要功能：

- **Webhook 集成**：促进自动订阅与 CloudWatch 告警关联的 AWS SNS 主题，确保 Keep 收到所有相关告警的通知。
- **支持自定义 SNS 主题**：允许使用现有的 SNS 主题并指定自定义 SNS 主题进行告警通知。
- **广泛的监控范围**：利用 CloudWatch 全面的告警系统监控应用程序和基础设施运行状况。
- **适应性身份验证**：适应永久和临时 AWS 凭据以满足各种安全和运营需求。

## 连接 Provider

要将 CloudWatch 与 Keep 集成，您需要：

- 一个具有访问 CloudWatch 和 SNS 服务权限的 AWS 账户。
- 配置了 API 访问的 Keep 账户。
- CloudWatch provider 的适当 AWS IAM 权限。

## 设置集成

<Tip>为确保顺利设置过程，请确保您的 AWS IAM 角色已正确配置了 CloudWatch 和 SNS 访问的必要权限。</Tip>

<AutoGeneratedSnippet />

### 步骤：

1. **配置 AWS IAM 角色**：确保 CloudWatch provider 使用的 IAM 角色具有 `cloudwatch:DescribeAlarms`、`cloudwatch:PutMetricAlarm`、`sns:ListSubscriptionsByTopic` 等相关操作的权限。
2. **指定身份验证详细信息**：在 Keep 平台中，在 CloudWatch provider 配置中输入 AWS Access Key、Secret 和 Region 详细信息。
3. **设置 SNS 主题（可选）**：如果使用自定义 SNS 主题，请在 provider 配置中指定其 ARN 或名称。Keep 将使用此主题来监听告警通知。
4. **激活 Provider**：在 Keep 中完成设置以开始接收 CloudWatch 告警。

## 故障排除

- 确保提供的 AWS 凭据具有正确且未过期的权限。
- 验证 SNS 主题已正确配置以向 Keep 发送通知。
- 检查 CloudWatch 告警以确保它们处于活跃状态并正确配置为在所需条件下触发。

## Webhook 集成修改

CloudWatch 的 webhook 集成将 Keep 添加为与 CloudWatch 告警关联的 SNS 主题的订阅者。此集成允许 Keep 接收 AWS 环境中触发的所有告警的通知。
该集成自动获取对 CloudWatch 中以下范围的访问权限：
- `cloudwatch:DescribeAlarms`
