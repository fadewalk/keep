## 概述

Grafana Loki 是一个日志聚合系统，设计用于存储和查询来自所有应用程序和基础设施的日志。入门的最简单方法是使用 Grafana Cloud，这是一个完全可组合的可观测性堆栈。

## 查询 Grafana Loki

Grafana Loki provider 允许您通过 `query` 和 `query_range` 类型查询 Grafana Loki 中的日志。以下是可用于查询的参数：

1. `query` 类型：

    - `query`：要执行的 [LogQL](https://grafana.com/docs/loki/latest/query/) 查询。不使用有效 LogQL 语法的请求将返回错误。
    - `limit`：返回的最大条目数。默认为 `100`。仅适用于产生流（日志行）响应的查询类型。
    - `time`：查询的执行时间，作为纳秒 Unix 时间戳或其他 [支持的格式](https://grafana.com/docs/loki/latest/reference/loki-http-api/#timestamps)。默认为 now。
    - `direction`：确定日志的排序顺序。支持的值为 `forward` 或 `backward`。默认为 `backward`。

2. `query_range` 类型：

    - `query`：要执行的 [LogQL](https://grafana.com/docs/loki/latest/query/) 查询。
    - `limit`：返回的最大条目数。默认为 `100`。仅适用于产生流（日志行）响应的查询类型。
    - `start`：查询的开始时间，作为纳秒 Unix 时间戳或其他 [支持的格式](https://grafana.com/docs/loki/latest/reference/loki-http-api/#timestamps)。默认为一小时前。Loki 返回时间戳大于或等于此值的结果。
    - `end`：查询的结束时间，作为纳秒 Unix 时间戳或其他 [支持的格式](https://grafana.com/docs/loki/latest/reference/loki-http-api/#timestamps)。默认为 now。Loki 返回时间戳小于此值的结果。
    - `since`：用于相对于 `end` 计算 `start` 的 `duration`。如果 `end` 在未来，`start` 计算为现在之前到此持续时间。为 `start` 指定的任何值都会覆盖此参数。
    - `step`：以 `duration` 格式或秒为单位的浮点数表示的查询分辨率步长宽度。`duration` 是形式为 `[0-9]+[smhdwy]` 的 Prometheus 持续时间字符串。例如，5m 表示 5 分钟的持续时间。默认为基于 `start` 和 `end` 的动态值。仅适用于产生矩阵响应的查询类型。
    - `interval`：仅返回在指定间隔或更大间隔的条目，可以是 `duration` 格式或秒为单位的浮点数。仅适用于产生流响应的查询。不要与 step 混淆，请参阅 [Step versus interval](https://grafana.com/docs/loki/latest/reference/loki-http-api/#step-versus-interval) 中的解释。
    - `direction`：确定日志的排序顺序。支持的值为 `forward` 或 `backward`。默认为 `backward`。
