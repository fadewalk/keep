## 概述

**AmazonSQS Provider** 便于：
- 将 SQS 消息作为告警进行消费
- 向 SQS 队列发送/推送通知消息

<AutoGeneratedSnippet />

## AmazonSQS Action 的输入

- `message`: str：通知的消息体/消息内容
- `group_id`: str | None：仅在队列类型为 FIFO 时强制填写，普通队列可忽略。
- `dedup_id`: str | None：仅在队列类型为 FIFO 时强制填写，普通队列可忽略。
- **kwargs: dict | None：您可以传递额外的键值对，这些键值对将作为 MessageAttributes 在通知中发送。

## AmazonSQS Action 的输出

更多详细信息，请访问 [sqs-documentation](https://boto3.amazonaws.com/v1/documentation/api/latest/reference/services/sqs/client/send_message.html#)。
   ```json
   {
        'MD5OfMessageBody': 'string',
        'MD5OfMessageAttributes': 'string',
        'MD5OfMessageSystemAttributes': 'string',
        'MessageId': 'string',
        'SequenceNumber': 'string'
   }
   ```

<Note>
  - 使用 AmazonSQS action 时，如果您的队列是 FIFO 队列，则**必须**传递 dedup_id 和 group_id。
  - MessageAttribute 中存在的所有额外字段都作为键值对字典存储在 alert.label 中。
  - 您可以在 SQS 队列消息中传递这些属性，Keep 将提取并使用这些字段用于告警：
    - name
    - status：可能的值 'firing' | 'resolved' | 'acknowledged' | 'suppressed' | 'pending'，默认为 'firing'。
    - severity：可能的值 'critical' | 'high' | 'warning' | 'info' | 'low'，默认为 'high'。
    - description

</Note>

<Note>
密钥对所需的权限：
1. AmazonSQSFullAccess：如果您想同时通知和接收，这是 sqs::read + sqs::write 范围。
2. AmazonSQSReadOnlyAccess：如果您只想接收，这是 sqs::read 范围。

您可以在以下位置找到这些权限：
IAM > 用户 > [YOUR_USER] > 权限 > 添加权限 > 添加权限 > 直接附加策略 > 搜索 SQS。

要创建密钥对，请按照以下步骤操作：
1. 在 AWS 控制台中搜索 IAM，按 Enter。
2. 进入用户页面。
3. 选择您想要创建的用户。
4. 点击 `创建访问密钥`。
5. 选择 `第三方服务`，点击 `下一步`。
6. 添加 `描述标签`，点击 `下一步`。
7. 复制/下载密钥对。
</Note>
