## 概述

Dash0 是现代化的 OpenTelemetry 原生可观测性平台，构建于 CNCF 开放标准之上，如 PromQL、Perses 和 OTLP，具有完整的成本控制功能。

## 将 Dash0 连接到 Keep

要将 Dash0 连接到 Keep，您需要在 Dash0 中创建一个 webhook。

1. 进入 Dash0 仪表板并点击组织设置。

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_1.png" />
</Frame>

2. 点击 `Notification Channels` 并创建一个类型为 `Webhook` 的新通知通道。

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_2.png" />
</Frame>

3. 为 webhook 命名，输入 [https://api.keephq.dev/alerts/event/dash0](https://api.keephq.dev/alerts/event/dash0) 作为 URL。

4. 按照以下步骤在 Keep 中创建一个新的 API 密钥。

5. 进入 Keep 仪表板，点击左下角的配置文件图标，然后点击 `设置`。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-1.png" />
</Frame>

6. 选择 `用户和访问` 选项卡，然后选择 `API 密钥` 选项卡，创建一个新的 API 密钥。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-2.png" />
</Frame>

7. 输入名称，选择角色为 `webhook`，然后点击 `创建 API 密钥`。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-3.png" />
</Frame>

8. 复制 API 密钥。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-4.png" />
</Frame>

9. 添加一个新的请求 header，键为 `X-API-KEY`，值为从 Keep 复制的 API 密钥，然后保存 webhook。

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_3.png" />
</Frame>

10. 进入 `Alerting` 下的 `Notifications`，如果需要，创建一个新的通知规则，或修改现有的通知规则以使用创建的 webhook。

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_4.png" />
</Frame>

11. 进入 `Alerting` 下的 `Checks`，创建一个新的检查或编辑现有检查以使用创建的通知规则。

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_5.png" />
</Frame>

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_6.png" />
</Frame>

12. 现在您将开始从 Dash0 在 Keep 中接收事件。
