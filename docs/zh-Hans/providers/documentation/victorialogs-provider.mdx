## 概述

VictoriaLogs 是来自 VictoriaMetrics 的开源用户友好的日志数据库。它针对高性能和低内存使用进行了优化。它可以处理高基数和高容量的日志。

注意：要添加认证，应配置 VMAuth。更多信息，请参阅 [VMauth 文档](https://docs.victoriametrics.com/vmauth/)。

### 无认证
- 无需其他参数，仅需要 `Grafana Loki Host URL`。

### HTTP 基本认证
- `HTTP basic authentication - Username`：用于 HTTP 基本认证的用户名。
- `HTTP basic authentication - Password`：用于 HTTP 基本认证的密码。

### Bearer
- `Bearer Token`：用于认证的 bearer 令牌。
- `X-Scope-OrgID Header`：用于 VictoriaLogs 多租户支持的组织 ID。（可选）

## 查询 VictoriaLogs

VictoriaLogs provider 允许您通过 `query`、`hits`、`stats_query` 和 `stats_query_range` 类型查询日志。以下是可用于查询的参数：

1. `query` 类型：

    - `query`：要执行的查询。
    - `limit`：返回的最大匹配条目数。
    - `timeout`：查询超时时间（秒）。
    - `AccountID`：用于 VictoriaLogs 的账户 ID。
    - `ProjectID`：用于 VictoriaLogs 的项目 ID。

2. `hits` 类型：

    - `query`：要执行的查询。
    - `start`：查询的开始时间。
    - `end`：查询的结束时间。
    - `step`：查询的步长。
    - `AccountID`：用于 VictoriaLogs 的账户 ID。
    - `ProjectID`：用于 VictoriaLogs 的项目 ID。

3. `stats_query` 类型：

    - `query`：要执行的查询。
    - `time`：查询的评估时间。

4. `stats_query_range` 类型：

    - `query`：要执行的查询。
    - `start`：查询的开始时间。
    - `end`：查询的结束时间。
    - `step`：查询的步长。
