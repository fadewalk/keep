## 连接 Provider

### API 密钥

要获取 Zabbix 认证令牌，请按照以下步骤操作，这些步骤分为三类（[文档](https://www.zabbix.com/documentation/current/en/manual/web_interface/frontend_sections/users/api_tokens)）：

首先，使用特权用户登录您的 Zabbix 账户（提供的 `zabbix_frontend_url`）。

#### 创建用户角色

1. 导航至 `Users` -> `User Roles` 部分。
2. 在屏幕右上角，点击 `Create user role`
3. 为角色提供一个指示性名称（例如 Keep Role）
4. 在 `User type` 选择框中，选择 `Super Admin`

- 这是因为我们需要的一些范围仅在 `Super Admin` 用户类型下可用。[见此处](https://www.zabbix.com/documentation/current/en/manual/api/reference/mediatype/create)

5. 从所有内容中删除所有复选框，只保留 1 个随机 `Access to UI elements`，这是任何角色都需要的。
6. 在 `API methods` 部分，选择 `Allow list` 并填写以下范围：
- `action.create`
- `action.get`
- `event.acknowledge`
- `mediatype.create`
- `mediatype.get`
- `mediatype.update`
- `problem.get`
- `script.create`
- `script.get`
- `script.update`
- `user.get`
- `user.update`

#### 创建用户

1. 导航至 `Users` -> `Users` 部分。
2. 按照说明添加新用户。为其提供一个指示性用户名（例如 KeepUser）
3. 在 `Permissions` 选项卡中，选择您刚刚创建的角色。
4. 点击 `Add`

#### 创建 API 令牌

1. 导航至 `Users` -> `API tokens` 部分。
2. 在屏幕右上角，点击 `Create API token`
3. 为 API 令牌提供一个指示性名称（例如 Keep Token）
4. 选择您刚刚创建的用户
5. 取消选择 `Set expiration date and time` 复选框，然后点击 `Add`
6. 复制生成的 API 令牌并在 Keep 中保留以供进一步使用。

## 注意事项

<Note>
  在安装 Zabbix webhook 时，Keep 会自动添加一个新的类型为 Keep 的媒体类型到您的媒体类型中。

  添加新媒体类型后，Keep
  会自动将此媒体类型作为媒体添加到所有现有用户，以获取
  来自 Zabbix 的所有传入告警。
</Note>

## Webhook 集成修改

自动 webhook 集成授予 Keep 对 Zabbix 实例内以下范围的访问权限：
- `mediatype.get`
- `mediatype.update`
- `mediatype.create`
- `user.get`
- `user.update`

您可以在 **Alerts > Media Types** 下查看 webhook 设置
